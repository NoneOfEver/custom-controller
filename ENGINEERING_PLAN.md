# Custom Controller — 工程计划（嵌入式 Rust / STM32F446RET6）

本文档用于指导后续所有实现工作（包含其它 agent）。硬件规格以 HARDWARE.md 为准。

## 0. 目标与范围

- 目标：为 STM32F446RET6 控制器建立可维护的 Rust 工程骨架，并实现以下外设的“驱动 + 数据采集 + 接口预留”
  - 4× 位置传感器 UART 接收（TTL，9600 8N1，DMA + IDLE 分帧）
  - 1× 上位机链路：USART1 → RS485（仅发送 TX，不接收）
  - 1× 摇杆三轴：ADC1 三通道扫描采样（IN8→IN4→IN5，软件触发；采样时间/滤波/标定后续可调整）
  - 5× 按键：周期扫描 + 20ms 去抖（板级上拉，低有效，MCU no-pull）
- 不在范围（由你后续补全，但要留接口）：
  - 位置传感器具体协议/字段/校验
  - 上位机协议（帧格式、发送频率、最大包长等）
  - 摇杆标定/映射参数与滤波策略

## 1. 关键技术决策（在写代码前先定）

### 1.1 运行时/并发模型（建议二选一）

- 方案 A：RTIC（中断驱动、可预测；适合 DMA+IDLE、定时扫描按键/ADC）
- 方案 B：Embassy（async/await；适合多串口并发与定时任务）

约束：你偏好“固定回调函数 + 静态入口”的直观写法，因此更推荐 RTIC 或 bare-metal + 中断回调，而不是到处传 user_ctx。

决策产物：确定采用 A/B，并写入本文件（后续不轻易更改）。

- 已选：方案 B（Embassy，async/await）

### 1.2 日志与 panic 策略

- Debug 阶段建议：defmt + probe-rs（或同类方案）
- Release 阶段：最小化 panic 处理（按项目需要）

## 2. 工程里程碑与交付物

### M1：工程骨架可编译/可下载

交付物：
- Cargo 工程可 `cargo build` 通过（目标为 thumbv7em-none-eabihf）
- linker/内存布局/启动入口就绪（能在板上跑到 main 并打出最小日志或点灯占位）

验收标准：
- 通过一次“烧录 + 上电运行”验证（至少证明程序在跑）

### M2：时钟与外设时基落地

交付物：
- 时钟配置：HSE=8MHz，系统主频设为该芯片规格允许的最大值
- 基础定时能力：用于按键扫描周期、ADC 采样节拍（具体周期可后调）

验收标准：
- 定时节拍稳定（可通过日志计数或 GPIO 翻转测量）

### M3：UART（4×传感器 RX + 1×RS485 TX）驱动骨架

交付物：
- 4 路 UART/USART（UART4/USART2/USART3/USART6）
  - RX DMA
  - IDLE 中断触发“取出本次帧数据”的入口
  - 将“原始帧 bytes”推送到上层解析接口（解析由你后续补充）
- 1 路 RS485（USART1）
  - 仅实现 TX 发送队列/发送 API
  - 明确不启用 RX

验收标准：
- 任一路串口可在空闲中断后拿到一段 buffer（即使暂时不解析）
- RS485 发送函数可稳定发出字节流（可用示波器/USB-RS485 验证）

### M4：ADC1 三通道采样模块（摇杆三轴）

交付物：
- ADC1 扫描采样 3 通道（PB0/PA4/PA5；IN8→IN4→IN5）
- 输出：
  - 原始 ADC 值（u16）
  - 可选：归一化到 [0,1] 或 [-1,1] 的“未标定”值（标定参数后续再补）

验收标准：
- 采样值随摇杆运动变化且稳定

### M5：按键扫描与去抖

交付物：
- 5 路按键输入（PC0/PC1/PC2/PC3/PB7）
- 周期扫描 + 20ms 去抖
- 输出：按下/抬起/长按（若需要）等事件的接口（最小实现可仅有稳定电平）

验收标准：
- 去抖有效，无明显抖动误触发

### M6：协议层接口与整合

交付物：
- 位置传感器解析接口：例如 `on_frame(port_id, bytes)`（具体签名以后按你偏好定）
- 上位机发送接口：例如 `send_to_host(msg)`（具体协议后续由你实现）
- 将传感器/摇杆/按键数据汇总为“应用层状态结构”，便于上位机发送或本地逻辑使用

验收标准：
- 代码结构清晰：驱动层不依赖协议细节；协议层可替换

## 3. 推荐代码结构（可在初始化阶段落地）

- src/
  - main.rs（启动/调度/顶层组装）
  - board.rs（时钟、GPIO、外设实例化）
  - uart/
    - mod.rs
    - sensor_ports.rs（4 路 RX DMA + IDLE 封装）
    - host_tx.rs（USART1 仅 TX）
  - adc/
    - mod.rs
    - joystick.rs
  - input/
    - mod.rs
    - buttons.rs
  - protocol/
    - mod.rs（仅接口/trait/占位，不写具体协议）
  - app/
    - mod.rs（应用层状态与整合）

说明：具体模块命名可调整，但需要保持“驱动层/协议层/应用层”三层解耦。

## 4. 风险与注意事项

- UART DMA + IDLE：需要明确 buffer 管理策略（双缓冲/环形缓冲/拷贝时机），避免丢帧与数据竞争
- 5 路串口并发：DMA stream/channel 分配可能受限，需要在实现时核对 MCU DMA 映射
- ADC 扫描：采样时间与触发策略未来可能变更，代码应保持可配置
- 按键：板级上拉且低有效，MCU 侧应使用 no-pull，避免双上拉导致电平异常
